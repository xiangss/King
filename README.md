### 项目结构
```
四大模块：首页---产品---借钱---信用---我的
项目说明：此项目是提供用户在app上面借贷，然后利用借贷的金额购买app上面的具有升值意义的产品
闭环效果：注册--登录--填写信息--获取信用分--借钱--申请（收藏）产品--订单购买--还款
其他功能：反馈意见，我的评价，借款问题，客服咨询....
```

### 项目遇到的问题
```
1.闭环功能衔接不顺畅
具体：在开始项目之前，成员之间虽然探讨了如何去开发，但是没有细致的探讨衔接过程的具体操作
例如：订单页面请求的数据，前端这边以为申请后就会返回一条数据，实则后台是在进入订单页面时候返回所有订单数据
2.使用仓库储存数据，一旦从后台请求数据，用户所做的其他有关数据的操作就都没了
解决：每次用户的操作，都让后台发送一条数据，在仓库中push进一个新的数组当中，
这样用户操作后的数据会一直存在，不会被新请求的数据覆盖掉
3.vant中的noticeBar通知栏插件引入不了，动画石实现不了
解决：至今未解决，直接放弃
4.从仓库中获取数据渲染到结构上有时候会有报错
解决：因为仓库中的数据是异步请求过来的，渲染的时候是undefined，所以从仓库中拿来的数据给一个初始值
5.数据库的部署....
```
